1/ The web is bifurcating.

For 30 years we had one web — built for human eyes. Now a second web is emerging: markdown, structured data, and llms.txt files that AI agents consume.

There's no browser for that second web. Until now.

Introducing Markdown Browser: https://markdownbrowser.com

2/ When an AI agent fetches a webpage, it gets nav bars, ads, cookie banners, tracking scripts. The actual content is buried.

Markdown Browser shows you exactly what's left when you strip a site down to what matters to an agent. Some sites hold up beautifully. Others are chaos held together by CSS.

3/ Sites are publishing /llms.txt files — sitemaps designed for AI. But there's been no way to browse them as a human.

Markdown Browser makes llms.txt navigable with a sidebar, like a dev portal for the machine-readable web. Plus devtools: token cost, response time, spec compliance.

4/ Here's what surprised me: markdown is a better reading experience.

No popups. No layout shift. No cookie dialogs. No autoplaying videos. No "subscribe" modals. Just text, links, and code blocks.

It's reader mode on steroids — except the content is *designed* to be read this way.

5/ Also shipping Markdown Form Links — interactive forms using just markdown syntax.

`[Search](/{page=search|images|videos|places|news|shopping|scholar}/{query})`

One line. A dropdown + text input. No JavaScript. Degrades gracefully everywhere. This is what agent-friendly interactivity looks like.
